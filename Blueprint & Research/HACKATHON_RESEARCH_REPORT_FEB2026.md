**3. Impact & Usefulness (15-25%)**

**4. Design & User Experience (10-20%)**

**6. Business Viability (5-10%)**
- Market potential

**‚ùå CRITICAL MISTAKES:**

   - Rule violation ‚Üí disqualification
   - Must start AFTER official start time
   - Use pre-hackathon for planning/design only

2. **Poor Video Presentation**
   - Unstable camera/audio
   - Reading from script (unengaging)
   - Exceeding time limit (cut off)
   - No demo footage (just slides)

3. **Incomplete GitHub Repo**
   - No README ‚Üí instant deduction
   - No license file ‚Üí cannot be open-sourced
   - No commit history (looks like copied code)
   - Dependencies not documented

4. **Ignoring Sponsor Technology**
   - Using generic solutions instead of sponsor APIs
   - Not leveraging unique sponsor features
   - Judges prioritize sponsor tech usage

5. **Scope Creep / Unfinished MVP**
   - Too ambitious ‚Üí broken demo
   - Better: Simple, working, polished than complex, broken
   - Minimum Viable Product must be functional

6. **Team Communication Issues**
   - Uneven contribution (some members do nothing)
   - No clear role division
   - Missing team member during pitch/Q&A

7. **Ignoring Judging Criteria**
   - Building something technically cool but irrelevant to theme
   - Not addressing all submission requirements
   - Missing mandatory components

8. **Poor Time Management**
   - Last-minute submission (technical issues)
   - No time for video editing/polishing
   - No buffer for unexpected problems
